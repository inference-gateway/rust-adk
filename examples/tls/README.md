# TLS / mTLS example

This scenario shows how to wire [`TlsConfig`](../../src/config.rs) through
the [`A2AServer`](../../src/server/server_core.rs) so the A2A endpoint
terminates TLS, and how to optionally require every client to present a
certificate signed by a trusted CA (mTLS) — i.e. the
[`MutualTlsSecurityScheme`](../../src/a2a_types.rs) the A2A spec
describes.

The TLS stack is `axum-server` + `rustls` (with the `ring` crypto
provider). Rustls was chosen over native-tls because (1) it is pure Rust
and avoids the OpenSSL toolchain on container builds, and (2) it gives
us programmatic access to the negotiated `ServerConnection`, which is
what makes the mTLS subject extraction below tractable.

## Layout

```
examples/tls/
├── certs/                  # generated by make-certs.sh (gitignored)
├── client/main.rs          # reqwest-based HTTPS client (TLS + mTLS)
├── docker-compose.yaml     # tls and mtls compose profiles
├── make-certs.sh           # openssl-based CA + server + client PKI
├── README.md               # you are here
└── server/main.rs          # A2AServer with TlsConfig from env
```

## Generate the example PKI

The example uses an isolated, self-signed PKI so it does not need any
real domain or CA. Generate it with:

```bash
./examples/tls/make-certs.sh
```

This produces:

- `certs/ca.crt` / `certs/ca.key` — root CA. Both the server (for its
  own chain) and the client (as a trust root) consume `ca.crt`. The CA
  also signs the client cert in mTLS mode.
- `certs/server.crt` / `certs/server.key` — leaf with
  `subjectAltName = DNS:localhost, IP:127.0.0.1`.
- `certs/client.crt` / `certs/client.key` — leaf with
  `CN=demo-client`, signed by `ca.crt`.

The certificates are gitignored; rerun the script if they expire or you
want to rotate them.

## Run the server (TLS only)

```bash
SERVER_TLS_ENABLE=true \
SERVER_TLS_CERT_PATH=examples/tls/certs/server.crt \
SERVER_TLS_KEY_PATH=examples/tls/certs/server.key \
PORT=8443 \
cargo run -p tls-server
```

Smoke-test with curl:

```bash
curl --cacert examples/tls/certs/ca.crt https://localhost:8443/health
curl --cacert examples/tls/certs/ca.crt https://localhost:8443/.well-known/agent.json
```

Both should return 200 OK.

## Run the server (mutual TLS)

Add `SERVER_TLS_CLIENT_CA_PATH` to require every client to present a
certificate signed by `ca.crt`:

```bash
SERVER_TLS_ENABLE=true \
SERVER_TLS_CERT_PATH=examples/tls/certs/server.crt \
SERVER_TLS_KEY_PATH=examples/tls/certs/server.key \
SERVER_TLS_CLIENT_CA_PATH=examples/tls/certs/ca.crt \
PORT=8443 \
cargo run -p tls-server
```

Now an unauthenticated client is rejected at the TLS layer:

```bash
$ curl --cacert examples/tls/certs/ca.crt https://localhost:8443/health
curl: (35) error:0A00045C:SSL routines::tlsv13 alert certificate required
```

A client that presents the demo cert succeeds:

```bash
curl --cacert examples/tls/certs/ca.crt \
     --cert examples/tls/certs/client.crt \
     --key  examples/tls/certs/client.key \
     https://localhost:8443/health
```

Server-side, the request is logged with the cert subject and the JSON-RPC
handler can extract the principal via
`axum::Extension<inference_gateway_adk::PeerCert>`:

```rust
use axum::Extension;
use inference_gateway_adk::PeerCert;

async fn my_handler(Extension(peer): Extension<PeerCert>) {
    if let Some(p) = peer.0 {
        println!("authenticated client: {}", p.subject);
    }
}
```

## Run the bundled client

The example client mirrors the curl commands above:

```bash
# TLS only
TLS_CA_PATH=examples/tls/certs/ca.crt \
SERVER_URL=https://localhost:8443 \
cargo run -p tls-client

# mTLS
TLS_CA_PATH=examples/tls/certs/ca.crt \
TLS_CLIENT_CERT_PATH=examples/tls/certs/client.crt \
TLS_CLIENT_KEY_PATH=examples/tls/certs/client.key \
SERVER_URL=https://localhost:8443 \
cargo run -p tls-client
```

## Environment knobs

| Variable | Purpose |
| --- | --- |
| `SERVER_TLS_ENABLE` | Required. Set to `true` to flip `A2AServer::serve` onto the TLS listener. |
| `SERVER_TLS_CERT_PATH` | PEM file with the server certificate chain. |
| `SERVER_TLS_KEY_PATH` | PEM file with the server private key (PKCS#1, PKCS#8, or SEC1). |
| `SERVER_TLS_CLIENT_CA_PATH` | Optional. When set, the server requires mTLS and trusts client certificates signed by any CA in this PEM bundle. |
| `PORT` | Listener port (default `8080`; the example sets `8443`). |
